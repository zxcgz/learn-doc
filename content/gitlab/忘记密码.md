如果忘了 GitLab 管理员（通常是 `root`）密码，最稳妥的办法是在服务器上用 **rake 任务**或 **Rails 控制台**重置。下面按安装方式给出步骤（任选其一即可）。

# 一、Omnibus 安装（含官方 Docker 镜像）

> 适用于用 apt/yum 安装的 `gitlab-ce/ee`，以及 `gitlab/gitlab-ce` / `gitlab/gitlab-ee` Docker 容器。

**方法 A：一条命令重置（推荐）**

```bash
# 物理机/VM 安装：
sudo gitlab-rake "gitlab:password:reset[root]"

# 如果是 Docker（容器名按实际替换）：
docker exec -it gitlab /bin/bash -lc 'gitlab-rake "gitlab:password:reset[root]"'
```

按提示输入新密码即可。若不确定用户名，把 `[root]` 去掉，它会给你列出用户清单供选择。

**方法 B：Rails 控制台手动改**

```bash
# 进入 Rails 控制台
sudo gitlab-rails console -e production
# Docker：
# docker exec -it gitlab /bin/bash -lc 'gitlab-rails console -e production'

# 在控制台内：
user = User.find_by_username('root') || User.admins.first
user.password = '你的新密码'
user.password_confirmation = '你的新密码'
user.password_automatically_set = false
user.save!
# 如被锁定：
user.unlock_access!    # 可选
# 如开启了 2FA，需要同时关掉（见下文）
```

# 二、源码安装（从源码部署）

```bash
cd /home/git/gitlab
sudo -u git -H bundle exec rake gitlab:password:reset RAILS_ENV=production
# 或进入控制台：
sudo -u git -H bundle exec rails console -e production
# 然后在控制台里用与上文相同的 Ruby 代码设置密码
```

# 三、开启了两步验证（2FA）也忘了

重置密码后如果仍然卡在 2FA，可在 **Rails 控制台**里禁用：

```ruby
user = User.find_by_username('root')
user.disable_two_factor!   # 关闭 2FA
user.save!
```

（老版本也可：`user.otp_required_for_login = false; user.save!`）

# 四、连管理员用户名都忘了

```bash
# 列出管理员用户名与邮箱
sudo gitlab-rails runner -e production "puts User.admins.active.pluck(:id,:username,:email).map{|x| x.join('  ')}"
```

# 五、想走“忘记密码”邮件但没配置 SMTP

1. 在 `/etc/gitlab/gitlab.rb`（Docker 在容器内相同路径）配置 SMTP，例如：
    
    ```ruby
    gitlab_rails['smtp_enable'] = true
    gitlab_rails['smtp_address'] = "smtp.example.com"
    gitlab_rails['smtp_port'] = 587
    gitlab_rails['smtp_user_name'] = "no-reply@example.com"
    gitlab_rails['smtp_password'] = "你的密码"
    gitlab_rails['smtp_domain'] = "example.com"
    gitlab_rails['smtp_authentication'] = "login"
    gitlab_rails['smtp_enable_starttls_auto'] = true
    gitlab_rails['gitlab_email_from'] = "no-reply@example.com"
    ```
    
2. 应用配置：
    
    ```bash
    sudo gitlab-ctl reconfigure
    sudo gitlab-ctl restart   # 可选
    ```
    
3. 登录页使用“Forgot your password?” 进行邮件重置。
    

# 六、常见坑位 & 排错

- **提示用户被锁定**：控制台执行 `user.unlock_access!`。
    
- **外部登录（LDAP/SAML/OAuth）**：`root` 用户通常是本地账户，不受外部身份源限制；其它管理员如果是外部账户，建议改密码走外部 IdP。
    
- **提示密码策略不通过**：GitLab 可能启用了复杂度或长度限制，设更强的新密码再 `user.save!`。
    
- **多节点/Geo**：在主节点上操作；从节点会同步。
    
- **无法进入容器**：先 `docker ps` 找容器名或 `podman ps`（若用 podman）。
