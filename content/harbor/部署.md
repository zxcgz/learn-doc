#  准备

1.  Harbor是运行在Docker上的服务，并且依赖docker-compose，所以需要确保服务器上具有Docker（20.10.10）和docker-compose（v1.18.0+或者docker compose v2）的依赖，
    
2.  使用docker命令push镜像时，默认需要docker仓库开启https，为了开启Https，需要安装openssl（版本越高越好）
    

# 下载

1.  在Github上下载**离线**版的安装包[Harbor](https://github.com/goharbor/harbor/releases)
    
2.  将安装包上传到服务器上（可以先根据文档验证签名），并解压 
    

# 证书

证书步骤并不是必须的，如果不配置证书则push镜像时需要在客户端做额外的配置

1.  使用OpenSSL生成证书
    
    1.  生成CA私钥
        

```java
openssl genrsa -out ca.key 4096
```

2.  生成CA证书
    

```java
openssl req -x509 -new -nodes -sha512 -days 3650 \
 -subj "/C=CN/ST=Shandong/L=Jinan/O=sinocipher/OU=Harbor/CN=Root CA" \
 -key ca.key \
 -out ca.crt
```

3.  生成服务器私钥
    

```java
openssl genrsa -out yourdomain.com.key 4096
```

4.  在同级目录下创建openssl.cnf文件（不确定是否必须）
    

```java
[req]
distinguished_name = req_distinguished_name
req_extensions = v3_req

[req_distinguished_name]
countryName = Country Name (2 letter code)
countryName_default = CH
stateOrProvinceName = State or Province Name (full name)
stateOrProvinceName_default = GD
localityName = Locality Name (eg, city)
localityName_default = ShenZhen
organizationalUnitName  = Organizational Unit Name (eg, section)
organizationalUnitName_default  = organizationalUnitName
commonName = Internet Widgits Ltd
commonName_max  = 64

[ v3_req ]
# Extensions to add to a certificate request
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names

[alt_names]

# 改成自己的ip
IP.1 = 172.16.24.143
IP.2 = 172.16.24.85
```

5.  生成服务器证书请求（如果没有生成openssl.cnf，则不需要`-config openssl.cnf`）
    

```java
openssl req -sha512 -new \
    -subj "/C=CN/ST=Shandong/L=Jinan/O=sinocipher/OU=Harbor/CN=yourdomain.com" \
    -key sinocipher.harbor.com.key \
    -out sinocipher.harbor.com.csr \
	-config openssl.cnf
```

6.  生成 x509 v3 扩展文件。（其中的IP.1部分可以扩展）
    

```java
cat > v3.ext <<-EOF
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[alt_names]
IP.1=192.168.7.114
EOF
```

7.  生成证书
    

```java
openssl x509 -req -sha512 -days 3650 \
    -extfile v3.ext \
    -CA ca.crt -CAkey ca.key -CAcreateserial \
    -in sinocipher.harbor.com.csr \
    -out sinocipher.harbor.com.crt
```

8.  将crt证书转换成cert
    

```java
openssl x509 -inform PEM -in sinocipher.harbor.com.crt -out sinocipher.harbor.com.cert
```

2.  向Docker和Harbor提供证书
    
    1.  将服务器证书和密钥复制到 Harbor 主机上的 certficates 文件夹中。如果没有相应的父目录需要创建，复制时需要使用cert证书
        

```java
cp sinocipher.harbor.com.cert /data/cert/
cp sinocipher.harbor.com.key /data/cert/
```

2.  将服务器证书、密钥和 CA 文件复制到 Harbor 主机上的 Docker 证书文件夹中。您必须先创建相应的文件夹。最后的目录是主机的ip地址
    

```java
cp sinocipher.harbor.com.cert /etc/docker/certs.d/192.168.7.114/
cp sinocipher.harbor.com.key /etc/docker/certs.d/192.168.7.114/
cp ca.crt /etc/docker/certs.d/192.168.7.114/
```

3.  重启Docker
    

```java
systemctl restart docker
```

# 部署

1.  检查。检查Harbor是否已经在运行，如果已经在运行并且需要重新部署开启Https，则需要关闭正在运行的容器，否则可以跳过部署操作
    

```java
docker-compose down -v
```

2.  编辑`harbor.yml`文件
    
    1.  首先将`harbor.yml.tmpl`重命名
        

```java
cp harbor.yml.tmpl harbor.yml
```

2.  使用编辑工具打开harbor.yml
    

```java
vim harbor.yml
```

3.  编辑`https`部分，将`certificate`和`private_key`部分替换成实际的证书、密钥的位置（如果配置了证书的话）
    

```java
# https related config
https:
  # https port for harbor, default is 443
  port: 443
  # The path of cert and key files for nginx
  certificate: /etc/docker/certs.d/192.168.7.114/sinocipher.harbor.com.cert
  private_key: /etc/docker/certs.d/192.168.7.114/sinocipher.harbor.com.key
```

4.  根据实际开启的端口情况，修改http和https部分的端口
    
5.  根据需要修改admin账户的密码以及其他配置
    
6.  保存`harbor.yml`文件
    
7.  在配置了证书的情况下，执行`prepare`
    

```java
./prepare
```

3.  部署
    
    1.  执行`install`安装harbor（如果之前已经安装过，但是修改过yml文件，则依旧需要在执行本节第一步的情况下执行部署步骤；如果没有修改yml文件则可以跳过此步骤）
        

```java
./install.sh
```

2.  启动服务（此步骤不能和上面同时进行）
    

```java
docker-compose up -d
```
