# 从零到一：可回滚部署脚本完整指南

> 本文适用于在 Linux 上用 **Bash** 实现“一键停服 → 备份 → 覆盖发布 → 启服”，并支持**失败自动回滚**与**手动回滚**。  
> **已屏蔽隐私**：示例中的数据库、口令、路径、服务名与文件名均使用占位符（如 `<DB_NAME>`、`<YOUR_APP_DIR>` 等），请按你的环境替换。

---

## 背景与目标

- 统一“备份 + 覆盖发布 + 回滚”的流程，减少人为失误
    
- 任何步骤失败可**自动回滚**，保障快速恢复
    
- 清单化（TSV）定义发布内容，便于审计与复现
    
- 支持 **MySQL/MariaDB** 逻辑备份（可关闭）
    
- 支持 **Jar 进程**或 **systemd 服务** 的停启控制
    

---

## 能力与流程概览

**`run` 流程**：  
停应用 → 数据库备份 → 目录打包 → 逐项复制（目标先备份）→ 启应用  
任何一步报错 ⇒ 自动回滚已复制的变更 + 重新启动旧版本

**`rollback` 流程**：  
停应用 → 按 Manifest 逆序恢复/删除 → 启应用

**`verify`**：核验 Manifest 中记录的单文件哈希

---

## 环境与依赖

- **Shell**：Bash 4+
    
- **通用命令**：`tar`、`gzip`（或 `pigz`）、`sha256sum`、`pgrep`、`install`、`flock`
    
- **数据库**（可选）：`mysql`、`mysqldump`（或 `mariadb-dump`）
    
- **进程控制**：
    
    - Jar 模式：`java`
        
    - systemd 模式：`systemctl`
        

> Debian/Ubuntu 安装示例：

```bash
sudo apt-get update
sudo apt-get install -y mysql-client tar gzip procps coreutils util-linux
sudo apt-get install -y pigz   # 可选：并行压缩
```

```

├─ copy-map.tsv                 # 发布清单（源<TAB>目标）
```

## 一步一步搭建

### 1) 创建目录

```bash
sudo mkdir -p /opt/deploy
cd /opt/deploy
```

### 2) 编写脚本（完整代码）

> 将下列**完整脚本**保存为 `/opt/deploy/deploy_with_backup.sh` 并 `chmod +x`。  
> 所有 `<>` 包裹的内容为**占位符**，请按实际环境替换。

```bash
#!/usr/bin/env bash
set -Eeuo pipefail

# ===========================================
# deploy_with_backup.sh (with Java stop/start & auto-rollback)
# 能力: 停应用 → 备份DB/目录 → 带备份复制 → 启应用；失败自动回滚并重启旧版本
# ===========================================

# ======== CONFIG（按需替换占位符）===========
BACKUP_ROOT="./cosign-backup"           # 备份根目录（可相对脚本）
LOCK_FILE="./tmp/deploy_with_backup.lock"

# 数据库（可关闭 DB 备份）
DB_ENABLED=1
DB_NAME="<DB_NAME>"
DB_HOST="<DB_HOST>"                      # 例如 127.0.0.1
DB_PORT="3306"
DB_USER="<DB_USER>"
DB_PASS="<DB_PASS>"                      # 建议按文末“安全建议”改为更安全方式

# 需要整体打包的目录（tar.gz）
BACKUP_DIRS=(/<YOUR_APP_DIR>)

# 拷贝清单（TSV：源<TAB>目标）
COPY_MAP="./copy-map.tsv"

# 应用控制：二选一
APP_MODE="jar"                           # "jar" 或 "systemctl"

# systemd 模式
APP_SERVICE="<YOUR_SERVICE>.service"

# jar 模式
JAVA_BIN="/usr/bin/java"
JAVA_OPTS="-Xms256m -Xmx256m -XX:+UseSerialGC"
APP_JAR="/<YOUR_APP_DIR>/<APP_JAR_NAME>.jar"   # 部署后的 Jar 路径
APP_ARGS=""
APP_PIDFILE="/<YOUR_APP_DIR>/app.pid"
APP_LOG="/<YOUR_APP_DIR>/app.out"
STOP_MATCH="$(basename "$APP_JAR")"      # 仅用 JAR 文件名匹配停止（如 your-app.jar）

# 压缩器（优先 pigz）
COMPRESSOR="$(command -v pigz >/dev/null 2>&1 && echo pigz || echo gzip)"
# ===========================================

# 路径统一解析为“相对脚本目录”
SCRIPT_DIR="$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")" && pwd -P)"
[[ "${BACKUP_ROOT}" != /* ]] && BACKUP_ROOT="${SCRIPT_DIR}/${BACKUP_ROOT}"
[[ "${COPY_MAP}"    != /* ]] && COPY_MAP="${SCRIPT_DIR}/${COPY_MAP}"
[[ "${LOCK_FILE}"   != /* ]] && LOCK_FILE="${SCRIPT_DIR}/${LOCK_FILE}"
mkdir -p "$(dirname "$LOCK_FILE")"

# 日志 & 工具
c_green="\033[32m"; c_yellow="\033[33m"; c_red="\033[31m"; c_end="\033[0m"
log()  { echo -e "${c_green}[*]${c_end} $*"; }
warn() { echo -e "${c_yellow}[!]${c_end} $*" >&2; }
err()  { echo -e "${c_red}[x]${c_end} $*" >&2; }
need() { command -v "$1" >/dev/null 2>&1 || { err "缺少命令: $1"; exit 1; }; }

DRY_RUN="${DRY_RUN:-0}"
CURRENT_CMD="${1:-}"

NOW="$(date +'%Y-%m-%d_%H-%M-%S')"
RUN_DIR="${BACKUP_ROOT}/${NOW}"
DB_DIR="${RUN_DIR}/db"
DIRS_DIR="${RUN_DIR}/dirs"
FILES_DIR="${RUN_DIR}/files"
META_DIR="${RUN_DIR}/_meta"
MANIFEST="${RUN_DIR}/manifest.tsv"
mkdir -p "$DB_DIR" "$DIRS_DIR" "$FILES_DIR" "$META_DIR"

sha256f()   { sha256sum "$1" | awk '{print $1}'; }
tsv_field() { printf "%s" "$1" | tr '\n' ' ' | tr '\t' ' '; }
manifest_header() {
  echo -e "action\tsrc\tdst\tbackup\tsha256_after\tnote" > "$MANIFEST"
}
append_manifest() {
  echo -e "$(tsv_field "$1")\t$(tsv_field "${2:-}")\t$(tsv_field "${3:-}")\t$(tsv_field "${4:-}")\t$(tsv_field "${5:-}")\t$(tsv_field "${6:-}")" >> "$MANIFEST"
}

detect_dump() {
  if command -v mysqldump >/dev/null 2>&1; then echo "mysqldump"
  elif command -v mariadb-dump >/dev/null 2>&1; then echo "mariadb-dump"
  else err "需要 mysqldump 或 mariadb-dump"; exit 1; fi
}

# =========== Java 控制 ===========
get_pids_by_match() {
  local pat="${1//./\\.}"             # 转义点号，避免正则过度匹配
  pgrep -f -- "java.*${pat}" || true
}
wait_pid_exit() {
  local pid="$1" timeout="${2:-30}" t=0
  while kill -0 "$pid" 2>/dev/null; do
    sleep 1; t=$((t+1)); [[ $t -ge $timeout ]] && return 1
  done
  return 0
}

stop_app() {
  log "停止应用（模式=${APP_MODE})..."
  [[ "$DRY_RUN" == "1" ]] && { log "(dry-run) 跳过停止应用"; return; }

  case "$APP_MODE" in
    systemctl)
      need systemctl
      systemctl stop "$APP_SERVICE" || true
      local t=0
      while systemctl is-active --quiet "$APP_SERVICE"; do
        sleep 1; t=$((t+1)); [[ $t -ge 30 ]] && { warn "systemctl stop 超时（30s）"; break; }
      done
      ;;
    jar)
      log "停止匹配关键字: $STOP_MATCH"
      local stopped=0
      if [[ -f "$APP_PIDFILE" ]]; then
        local pid; pid="$(cat "$APP_PIDFILE" 2>/dev/null || true)"
        if [[ -n "${pid:-}" ]] && kill -0 "$pid" 2>/dev/null; then
          log "通过 PIDFILE 终止 PID=$pid"
          kill -TERM "$pid" || true
          wait_pid_exit "$pid" 20 || kill -KILL "$pid" || true
          stopped=1
        fi
      fi
      local pids; pids="$(get_pids_by_match "$STOP_MATCH")"
      if [[ -n "${pids:-}" ]]; then
        log "按关键字匹配终止: $pids"
        kill -TERM $pids 2>/dev/null || true
        sleep 2
        for p in $pids; do wait_pid_exit "$p" 20 || kill -KILL "$p" 2>/dev/null || true; done
        stopped=1
      fi
      [[ $stopped -eq 1 ]] && log "应用已停止" || log "未发现需要停止的进程"
      ;;
    *) err "未知 APP_MODE: $APP_MODE"; exit 1;;
  esac
}

start_app() {
  log "启动应用（模式=${APP_MODE})..."
  [[ "$DRY_RUN" == "1" ]] && { log "(dry-run) 跳过启动应用"; return; }

  case "$APP_MODE" in
    systemctl)
      need systemctl
      systemctl start "$APP_SERVICE"
      systemctl is-active --quiet "$APP_SERVICE" || { err "systemctl 启动失败: $APP_SERVICE"; exit 1; }
      ;;
    jar)
      mkdir -p "$(dirname "$APP_LOG")" "$(dirname "$APP_PIDFILE")"
      nohup "$JAVA_BIN" $JAVA_OPTS -jar "$APP_JAR" $APP_ARGS >>"$APP_LOG" 2>&1 &
      echo $! > "$APP_PIDFILE"
      sleep 1
      local pid; pid="$(cat "$APP_PIDFILE" 2>/dev/null || true)"
      if [[ -z "${pid:-}" ]] || ! kill -0 "$pid" 2>/dev/null; then
        err "Java 应用启动失败（未获取有效 PID）。请检查 $APP_LOG"
        exit 1
      fi
      log "应用已启动，PID=$pid"
      ;;
    *) err "未知 APP_MODE: $APP_MODE"; exit 1;;
  esac
}
# =====================================

backup_database() {
  [[ "${DB_ENABLED}" == "1" ]] || { warn "已禁用数据库备份"; return; }
  [[ -n "${DB_NAME}" ]] || { warn "未配置 DB_NAME，跳过数据库备份"; return; }
  local DUMP_BIN; DUMP_BIN="$(detect_dump)"
  local outfile="${DB_DIR}/${DB_NAME}.sql"

  log "检查数据库连接(host=${DB_HOST} port=${DB_PORT} user=${DB_USER})..."
  if [[ "$DRY_RUN" != "1" ]]; then
    MYSQL_PWD="$DB_PASS" mysql --protocol=TCP -h"$DB_HOST" -P"$DB_PORT" -u"$DB_USER" -e "SELECT 1" >/dev/null \
      || { err "数据库连接失败。请检查主机/端口/账号与网络连通性。"; exit 1; }
  fi

  log "数据库备份 -> ${outfile}.gz (${DUMP_BIN})"
  if [[ "$DRY_RUN" == "1" ]]; then
    append_manifest "db_backup" "$DB_NAME" "" "${outfile}.gz" "" "dry-run"; return
  fi

  if ! MYSQL_PWD="$DB_PASS" ${DUMP_BIN} \
        --host="$DB_HOST" --port="$DB_PORT" --user="$DB_USER" \
        --single-transaction --routines --triggers --events \
        --hex-blob --databases "$DB_NAME" > "$outfile"; then
    rm -f "$outfile"; err "数据库备份失败"; exit 1
  fi

  if ! "$COMPRESSOR" -f "$outfile"; then
    rm -f "$outfile" "${outfile}.gz" || true
    err "压缩数据库备份失败"; exit 1
  fi

  append_manifest "db_backup" "$DB_NAME" "" "${outfile}.gz" "" ""
}

backup_dirs() {
  ((${#BACKUP_DIRS[@]})) || { warn "未配置 BACKUP_DIRS，跳过目录备份"; return; }
  need tar; need "$COMPRESSOR"
  for d in "${BACKUP_DIRS[@]}"; do
    if [[ ! -d "$d" ]]; then warn "目录不存在: $d（跳过）"; continue; fi
    local base; base="$(echo "$d" | sed 's#/#_#g' | sed 's/^_//')"
    local outfile="${DIRS_DIR}/${base}.tar.gz"
    log "目录备份 -> $outfile  (from $d)"
    if [[ "$DRY_RUN" == "1" ]]; then
      append_manifest "dir_backup" "$d" "" "$outfile" "" "dry-run"
    else
      tar -C "/" -cpf - "${d#/}" | "$COMPRESSOR" -c > "$outfile"
      append_manifest "dir_backup" "$d" "" "$outfile" "" ""
    fi
  done
}

copy_dir_contents() {
  local SRC="$1" DST="$2"
  mkdir -p "$DST"
  tar -C "$SRC" -cpf - . 2>/dev/null | tar -C "$DST" -xpf - 2>/dev/null || true
}

copy_files_with_backup() {
  [[ -f "${COPY_MAP}" ]] || { warn "COPY_MAP 不存在: ${COPY_MAP}，跳过文件复制"; return; }
  need tar; need "$COMPRESSOR"
  log "处理拷贝清单: $COPY_MAP"
  while IFS=$'\t' read -r SRC DST; do
    [[ -z "${SRC:-}" || -z "${DST:-}" ]] && continue
    [[ "${SRC:0:1}" == "#" ]] && continue
    if [[ ! -e "$SRC" ]]; then err "源不存在: $SRC"; exit 1; fi
    local R_SRC R_DST; R_SRC="$(realpath -m "$SRC")"; R_DST="$(realpath -m "$DST")"
    mkdir -p "$(dirname "$R_DST")"

    local backup_path=""
    if [[ -e "$R_DST" ]]; then
      local base; base="$(echo "$R_DST" | sed 's#/#_#g' | sed 's/^_//')"
      backup_path="${FILES_DIR}/before_${base}.tar.gz"
      log "备份将被覆盖的目标 -> $backup_path (from $R_DST)"
      if [[ "$DRY_RUN" != "1" ]]; then
        tar -C "/" -cpf - "${R_DST#/}" | "$COMPRESSOR" -c > "$backup_path"
      fi
      append_manifest "file_backup_before" "$R_DST" "" "$backup_path" "" ""
    fi

    log "复制: $R_SRC -> $R_DST"
    local sha=""
    if [[ "$DRY_RUN" != "1" ]]; then
      if [[ -d "$R_SRC" ]]; then
        copy_dir_contents "$R_SRC" "$R_DST"
      else
        install -m 0644 "$R_SRC" "$R_DST"
        sha="$(sha256f "$R_DST")"
      fi
    fi
    append_manifest "copy" "$R_SRC" "$R_DST" "$backup_path" "${sha}" ""
  done < <(grep -vE '^\s*#' "${COPY_MAP}" | sed '/^\s*$/d')
}

latest_manifest() {
  local latest_link="${BACKUP_ROOT}/latest/manifest.tsv"
  if [[ -f "$latest_link" ]]; then echo "$latest_link"; return 0; fi
  local latest_dir
  latest_dir="$(find "$BACKUP_ROOT" -mindepth 1 -maxdepth 1 -type d -printf '%T@ %p\n' | sort -nr | awk 'NR==1{print $2}')"
  [[ -n "${latest_dir:-}" && -f "${latest_dir}/manifest.tsv" ]] && { echo "${latest_dir}/manifest.tsv"; return 0; }
  return 1
}

do_rollback() {
  local m="${1:-}"
  if [[ -z "$m" ]]; then
    m="$(latest_manifest)" || { err "找不到可用的 manifest；请手动指定路径"; exit 1; }
  fi
  [[ -f "$m" ]] || { err "manifest 不存在: $m"; exit 1; }

  stop_app
  log "开始回滚，读取: $m"

  mapfile -t lines < <(tail -n +2 "$m")
  for (( idx=${#lines[@]}-1 ; idx>=0 ; idx-- )); do
    IFS=$'\t' read -r action src dst backup sha note <<<"${lines[$idx]}"
    case "$action" in
      copy)
        if [[ -n "${backup:-}" && -f "${backup}" ]]; then
          log "回滚文件: $dst  <-  ${backup}"
          if [[ -e "$dst" ]]; then
            if [[ -d "$dst" ]]; then rm -rf "$dst"; else rm -f "$dst"; fi
          fi
          "$COMPRESSOR" -d -c "${backup}" | tar -C / -xpf -
        else
          if [[ -e "$dst" ]]; then
            log "移除新文件(无备份): $dst"
            if [[ -d "$dst" ]]; then rm -rf "$dst"; else rm -f "$dst"; fi
          fi
        fi
        ;;
    esac
  done
  log "回滚完成（文件部分）。目录/数据库如需整体恢复请谨慎手动执行。"
  start_app
}

verify_manifest_targets() {
  local m; m="$(latest_manifest)" || { err "没有可用 manifest"; exit 1; }
  log "核验清单（单文件 SHA256）：$m"
  local ok=0 bad=0
  while IFS=$'\t' read -r action src dst backup sha note; do
    if [[ "$action" == "copy" && -n "${sha:-}" && -f "$dst" ]]; then
      cur="$(sha256f "$dst")"
      if [[ "$cur" == "$sha" ]]; then echo "OK  $dst"; ((ok++)) || true
      else echo "BAD $dst (expect $sha got $cur)"; ((bad++)) || true
      fi
    fi
  done < <(tail -n +2 "$m")
  echo "Summary: OK=$ok BAD=$bad"
}

# 运行时错误 → 自动回滚（仅 run）
on_error() {
  local line="${1:-?}"
  err "发生错误（第 $line 行）。尝试回滚并重启应用。"
  if [[ "${CURRENT_CMD}" == "run" && -f "$MANIFEST" ]]; then
    do_rollback "$MANIFEST" || warn "自动回滚失败，请手动检查。"
  fi
  exit 1
}
trap 'on_error $LINENO' ERR

main_run() {
  need "$COMPRESSOR"; need tar
  manifest_header
  log "输出目录: $RUN_DIR"

  stop_app
  backup_database
  backup_dirs
  copy_files_with_backup
  ln -sfn "$RUN_DIR" "${BACKUP_ROOT}/latest"
  start_app

  log "完成。回滚清单：${MANIFEST}"
  log "回滚最近一次：./deploy_with_backup.sh rollback"
}

# 并发锁，避免同时执行
exec 9>"$LOCK_FILE" || true
flock -n 9 || { err "已有实例在运行（锁: $LOCK_FILE）"; exit 1; }

CMD="${1:-}"
case "$CMD" in
  run)       main_run ;;
  rollback)  do_rollback "${2:-}" ;;
  verify)    verify_manifest_targets ;;
  ""|help|-h|--help)
    cat <<'USAGE'
用法：
  ./deploy_with_backup.sh run           # 停应用→备份→复制→启应用
  ./deploy_with_backup.sh rollback      # 停应用→按 latest 回滚→启应用
  ./deploy_with_backup.sh rollback /path/to/manifest.tsv
  ./deploy_with_backup.sh verify        # 校验最新清单中的单文件哈希
可选：
  DRY_RUN=1 ./deploy_with_backup.sh run # 演练模式（不落地，不启停）
USAGE
    ;;
  *) err "未知子命令: $CMD（可选 run|rollback|verify）"; exit 1;;
esac
```

### 3) 准备拷贝清单 `copy-map.tsv`

> 格式：**每行两列**，`源路径<TAB>目标路径`（必须是 **Tab**，不是空格）

示例（将线上旧 Jar 复制为带日期名 → 部署新 Jar）：

```
/<YOUR_APP_DIR>/<APP_JAR_NAME>.jar	/<YOUR_APP_DIR>/<APP_JAR_NAME>-<DATE>.jar
./build/<APP_JAR_NAME>.jar	        /<YOUR_APP_DIR>/<APP_JAR_NAME>.jar
```

示例（覆盖目录内容）：

```
./config/	                        /<YOUR_APP_DIR>/config/
```

自检：

```bash
cat -A copy-map.tsv | sed -n '1,10p'       # 列间应显示 ^I（Tab）
awk -F'\t' 'NF!=2{print "格式有误：第",NR,"行 ->",$0}' copy-map.tsv
sed -i 's/\r$//' copy-map.tsv              # 如为 CRLF，转换为 LF
```

### 4) 按需配置（Jar 或 systemd）

- **Jar 模式（默认）**  
    填好 `APP_JAR`、`JAVA_OPTS`、`APP_ARGS`、`APP_LOG`、`APP_PIDFILE`；  
    停止匹配使用 **文件名**（`STOP_MATCH="$(basename "$APP_JAR")"`），避免路径差异。
    
- **systemd 模式**  
    将 `APP_MODE="systemctl"` 并设置 `APP_SERVICE="<YOUR_SERVICE>.service"`。  
    （若需要，创建服务单元并 `systemctl daemon-reload`）

## 如何使用

### 演练（Dry-Run）

不落地改动，不启停应用，验证流程与清单是否正确。

```bash
cd /opt/deploy
DRY_RUN=1 ./deploy_with_backup.sh run
```

### 正式发布

```bash
cd /opt/deploy
./deploy_with_backup.sh run
```

- 会停应用 → 备份 DB/目录 → 覆盖复制（目标先备份）→ 启应用
    
- 任一步失败：自动回滚**已复制的文件**，并重启旧版本
    

### 回滚（最近一次/指定一次）

```bash
# 回滚到 latest（软链指向上一次发布）
./deploy_with_backup.sh rollback

# 回滚到指定 manifest
./deploy_with_backup.sh rollback <BACKUP_ROOT>/YYYY-MM-DD_HH-MM-SS/manifest.tsv
```

### 校验单文件哈希

```bash
./deploy_with_backup.sh verify
```

## 快照输出结构与含义

```
<BACKUP_ROOT>/YYYY-MM-DD_HH-MM-SS/
├─ db/                 # 数据库备份（*.sql.gz）
├─ dirs/               # 目录打包（*.tar.gz）
├─ files/              # 覆盖前的目标备份（before_*.tar.gz）
├─ _meta/              # 预留元信息
└─ manifest.tsv        # 回滚清单（动作明细）
```

并更新软链：`<BACKUP_ROOT>/latest -> 本次目录`

## Manifest 字段说明

`manifest.tsv`（TSV 格式）列：

- `action`：`db_backup | dir_backup | file_backup_before | copy`
    
- `src`：源对象（DB 名/源路径）
    
- `dst`：目标路径（仅 `copy` 有值）
    
- `backup`：备份路径（`*.tar.gz` 或 `*.sql.gz`）
    
- `sha256_after`：单文件复制后的哈希（目录留空）
    
- `note`：备注
    

> 回滚时仅对 `copy` 逐条逆序处理：
> 
> - 有 `backup` ⇒ 用备份覆盖回去
>     
> - 无 `backup` ⇒ 删除新建目标
>

## 常见问题与排错

### 拷贝没执行？

- `COPY_MAP` 不是 Tab 分隔或为 CRLF：见上文自检命令
    
- `COPY_MAP` 路径不正确（脚本解析为**相对脚本目录**）
    
- 源路径不存在：脚本会报错中止（查看终端输出）
    

### 停止进程未命中？

- 确认 `STOP_MATCH` 为 **JAR 文件名**（如 `your-app.jar`）
    
- 自检：
    
    ```bash
    pgrep -fa -- "java.*your-app\.jar"
    ```
    

### 启动失败？

- 查看 `APP_LOG`（如 `/.../app.out`）
    
- 检查端口占用、配置、依赖
    

### 数据库备份失败？

- 账号/口令/网络/权限
    
- 连通性：
    
    ```bash
    mysql -h<DB_HOST> -P<DB_PORT> -u<DB_USER> -p'<DB_PASS>' -e "SELECT 1"
    ```

## 安全与合规建议

- **口令不建议明文**写在脚本：
    
    - 可改为读取权限为 600 的配置文件，或通过环境变量（例如 `MYSQL_PWD`）注入
        
    - 避免将口令出现在命令行参数（可被 `ps` 看到）
        
- 限制脚本、日志与备份目录的权限（仅管理员可读写）
    
- 大库/热库：逻辑备份耗时较长，尽量在低峰或只做冷备
    
- 回滚目录/数据库：**谨慎**操作，避免误覆盖（脚本默认不自动恢复 DB/dirs）

## 日常运维与清理

仅保留最近 10 次快照（示例）：

```bash
ls -1dt "<BACKUP_ROOT>/"* | tail -n +11 | xargs -r rm -rf
```

可写入 crontab（每天凌晨 3:30 清理一次）：

```bash
30 3 * * * ls -1dt "<BACKUP_ROOT>/"* | tail -n +11 | xargs -r rm -rf
```

## 附录：按需裁剪/扩展

- **关闭 DB 备份**：`DB_ENABLED=0`
    
- **仅拷贝不备份**：可自行删除 `backup_dirs` 或 `backup_database` 调用
    
- **Jar 可执行位/属主**：把 `install -m 0644` 调整为 `install -m 0755 -o <USER> -g <GROUP>`
    
- **支持 rsync**：目录复制改为 `rsync -a --delete "$R_SRC/" "$R_DST/"`（语义不同，请评估）
    
- **多环境配置**：提取 `CONFIG` 到外部 `*.conf`，用 `source` 按环境加载
    
- **其它数据库**：可按同样结构包装 `pg_dump` / `mariabackup` 等
